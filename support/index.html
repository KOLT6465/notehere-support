NoteHere Support

Help for Safari and Chrome

Enable NoteHere In Safari

  1. Open the NoteHere app.
  2. Choose Open Safari Settings.
  3. Enable NoteHere under Extensions. Safari adds NoteHere's button to the toolbar automatically.
  4. Allow access to websites when Safari asks.
  5. On a normal webpage, open NoteHere's toolbar menu and choose New Note.

If you previously removed the button, choose View > Customize Toolbar in Safari and drag NoteHere back to the toolbar. Safari controls toolbar customization, so NoteHere cannot override a button the user removed.

Enable NoteHere In Chrome

  1. Install NoteHere, then open Chrome's Extensions menu from the puzzle-piece button.
  2. Pin NoteHere so its button stays visible in the toolbar.
  3. Open a normal webpage whose address starts with http:// or https://.
  4. Select NoteHere and choose New Note. Your note saves automatically.
  5. Open Settings to search your unlimited local library, export or import a backup, and turn sound effects on or off.

If NoteHere was just installed or updated while a webpage was already open, refresh that page once before creating a note.

Protected Pages

Safari and Chrome do not permit extensions to run on browser settings, start or new-tab pages, extension stores, or some protected internal pages. NoteHere works on normal HTTP and HTTPS webpages.

Backups, Privacy, And Deletion

Notes stay locally on your device. In Chrome, use NoteHere Settings to export a complete backup or readable text copy, import a backup, search the library, and delete individual notes. In either browser, use the toolbar menu to clear notes from the current exact page. Safari users can remove all website notes from the NoteHere manager. Read the Privacy Policy.

If A Note Does Not Appear

  1. Refresh the webpage once.
  2. Confirm that its full address—including the path, query, and fragment—matches the page where you created the note.
  3. Open NoteHere and choose Show All Notes in case the note was hidden.

Get Help

Report a problem at GitHub Support Issues. Include your operating-system version, browser and browser version, the type of webpage, and steps to reproduce. Do not include private note text or confidential URLs.
